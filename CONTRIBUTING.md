## Contributing

Thanks for taking the time to contribute.

### Ways to help

- Improve documentation (README clarity, screenshots, usage notes)
- Refactor code for readability and correctness
- Add small, focused improvements to the crypto “lab” experience (e.g., showing Base64/hex outputs, better error messages)
- Fix bugs and add basic tests where practical

### Development setup

- Open the repository root in Android Studio.
- Let Gradle sync finish.
- Run the `app` configuration on an emulator or device.

### Pull requests

- Keep PRs small and focused.
- Use clear commit messages.
- Describe the **why** (not just the what) in the PR description.
- If you change behavior, include a short **test plan** (what you ran / clicked).
